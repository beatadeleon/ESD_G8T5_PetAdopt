How to use Github

  1. Do not commit and push changes on the main branch
  2. Create a new branch when editing files from the main branch
  3. Publish your remote branch on Github
  4. Commit changes to your remote branch, name the file edited and the changes made
  5. Once done, make a pull request for your remote branch to be merged onto the main branch
  6. If there are no conflicts, you may merge your remote branch into the main
  7. Once successful, you can delete your remote branch, fetch origin from the newly updated main branch
